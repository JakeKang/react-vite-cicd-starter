## 🌿 Git Branch 전략 가이드 (수정 완료)

### docs/11-Git-Branch-전략-완벽-가이드.md

#### 수정 사항 (v5.0 → v5.1)

**1. 제목 불일치 수정** ✅
```diff
- ## 2. 초보자에게 추천하는 전략: GitHub Flow
+ ## 2. 초보자에게 추천하는 전략: GitLab Flow
```

**2. 용어 통일** ✅
```diff
- Pull Request (PR)
+ Merge Request (MR)
```

**설명 추가**:
```
GitLab에서는 "Merge Request"라고 부릅니다.
(GitHub의 "Pull Request"와 동일한 개념)
```

#### 왜 이 수정이 중요한가?

**문제**:
- 목차: "GitHub Flow"
- 내용: "GitLab Flow"
- → 독자 혼란 유발

**해결**:
- 목차와 내용 일치
- 명확한 용어 사용
- GitLab 공식 용어 준수
